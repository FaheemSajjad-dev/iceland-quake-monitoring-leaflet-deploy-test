Current local/deploy workflow:

1. Make and verify changes in F:\iceland-quake-monitoring-leaflet.
2. Copy the same relevant changes to F:\iceland-quake-monitoring-leaflet-deploy-test.
3. Mirror changed files to:
   - G:\iceland-quake-monitoring-leaflet-recovered
   - G:\iceland-quake-monitoring-leaflet-deploy-test-recovered
4. Run tests/builds locally.
5. Commit and push both F repositories to GitHub.
6. Upload deploy-version changes to Pluto.
7. On Pluto:

   cd ~/iceland-quake
   ./deploy.sh

Current map/app state:

- Base layer label is Map.
- Earthquake markers use SVG Leaflet L.circleMarker rendering.
- Separate invisible hit targets make earthquake markers easier to click or tap without increasing the visible marker size.
- Volcano overlay opens a right-side panel.
- About button, distance scale, and faults legend shift left when the right panel is open.
- Faults overlay uses EGDI/HIKE WFS onshore Iceland records.
- Faults are fetched on first use and cached in memory for later toggles.
- Fault legend uses solid red for faults and dotted red for fissures.
- Lat/lon grid uses Iceland-focused spacing with one-decimal labels.
- Lat/lon labels are slightly bolder while the faults overlay is visible.
- Main latitude labels are shown; extra latitude midlines are drawn unlabeled.
- Longitude labels are anchored near the bottom with spacing to avoid overlap.
- Earthquake and volcano info cards open near the upper-left map work area.
- Magnitude scale focus ring has been suppressed.
- Bottom-right map attribution is compact and shows only the active basemap providers, plus EGDI/HIKE when faults are visible.
- Default Map layer uses MapLibre GL over OpenFreeMap; if MapLibre/WebGL initialization fails, the app falls back to a CARTO light raster basemap.
- Heatmap weights are density-first: M 3-4 = 0.20, M 4-5 = 0.30, M 5+ = 0.45.
- Flask-Limiter rate limiting is applied to public read/export endpoints with environment-configurable limits.
- Frontend audit, lint, build, and unit test checks are expected to pass before changes are copied to deploy-test.

Known browser note:

- The Map base layer uses MapLibre GL and requires WebGL.
- If Chrome on a MacBook has WebGL disabled by hardware acceleration settings, browser policy, or graphics state, MapLibre can fail to initialize in that browser.
- If MapLibre cannot initialize, the app falls back to a CARTO raster basemap while keeping Leaflet overlays usable.

Deployment notes:

- Pluto uses ~/iceland-quake as the active project folder.
- deploy.sh installs dependencies, builds frontend assets, stops the old Gunicorn process, and starts the backend on port 6000.
- Root-level deploy files such as deploy.sh, stop.sh, render.yaml, package files, and README files should be uploaded from the deploy F project when changed.
